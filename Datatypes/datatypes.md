| Category                | C Data Types                     | C++ Data Types                                      | Python Data Types                    | Detailed Explanation                                                                                                            |
| ----------------------- | -------------------------------- | --------------------------------------------------- | ------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------- |
| **Basic / Fundamental** | `int`, `char`, `float`, `double` | `int`, `char`, `float`, `double`, `bool`, `wchar_t` | `int`, `float`, `complex`, `bool`    | Basic building blocks. C++ adds `bool` and wide char support. Python supports arbitrary precision integers and complex numbers. |
| **Derived Types**       | Array, Pointer                   | Array, Pointer, Reference                           | List, Tuple, Range                   | C/C++ derived types are memory-based. Python uses high-level containers with built-in functionality.                            |
| **User-defined Types**  | `struct`, `union`, `enum`        | `class`, `struct`, `union`, `enum`                  | Classes, Objects                     | C++ introduces full OOP with classes. Python uses everything as objects.                                                        |
| **Enumeration**         | `enum`                           | `enum`, `enum class`                                | Not explicit (can use classes/dicts) | Used for named constants. C++ improves type safety with `enum class`.                                                           |
| **Void Type**           | `void`                           | `void`                                              | `NoneType` (`None`)                  | Represents absence of value. Python uses `None` instead of void.                                                                |
| **Boolean Type**        | Not built-in (uses int)          | `bool`                                              | `bool`                               | C treats 0 as false, non-zero as true. C++ and Python have explicit boolean types.                                              |
| **String Type**         | `char[]`                         | `string` (STL)                                      | `str`                                | Python has built-in string type. C uses character arrays. C++ provides `std::string`.                                           |
| **Set Types**           | Not available                    | Not directly (STL: `set`)                           | `set`, `frozenset`                   | Python has built-in support for unique collections.                                                                             |
| **Mapping Type**        | Not available                    | `map` (STL)                                         | `dict`                               | Python dictionaries are highly optimized key-value stores.                                                                      |
| **Memory Management**   | Manual (`malloc`, `free`)        | Manual + Semi-automatic (`new`, `delete`)           | Automatic (Garbage Collection)       | Python handles memory automatically, unlike C/C++.                                                                              |
| **Pointers**            | Yes                              | Yes                                                 | Not explicit                         | Python hides pointers but internally uses references.                                                                           |
| **References**          | No                               | Yes (`&`)                                           | Implicit references                  | C++ references act as aliases. Python variables are references to objects.                                                      |
| **Dynamic Typing**      | No                               | No                                                  | Yes                                  | Python variables can change type at runtime.                                                                                    |
| **Type Safety**         | Low                              | High                                                | Very High                            | Python prevents many type errors at runtime.                                                                                    |
| **Complex Numbers**     | Not supported                    | Not built-in (library)                              | `complex`                            | Python directly supports complex arithmetic.                                                                                    |
